# 채널 전략 설계 문서 (Standard)

> 새 keywords_input.csv를 넣어도 같은 품질로 3채널 초안이 재현되도록, 채널별 규칙과 type별 강조 매핑을
> 마케팅·원장 관점 근거와 함께 명문화한다. 이 문서가 `/insight`·`/generate`가 따르는 실제 규칙집이다.

---

## 1. 파이프라인 구조 — 새 CSV를 넣으면 일어나는 일

```
새 keywords_input.csv
     │
     ▼
① scripts/clean_keywords.py   ── 중복 제거 · tone_hint 정규화 · detail 결측 보완 · type 분포 집계
     │  (결정론적 스크립트, LLM 불필요 → 실행할 때마다 같은 입력엔 같은 출력)
     ▼
② /insight  (Claude Code 스킬)  ── 아래 2·3장의 채널 규칙 + type 매핑을 적용해 케이스별 전략 확정
     ▼
③ /generate (Claude Code 스킬)  ── 케이스마다 블로그·인스타·카카오 3채널 초안 생성
     ▼
④ /review   (Claude Code 스킬)  ── 글자 수·톤 차별화 자가 채점, 미달 시 보완
     ▼
output/blog_draft.md · instagram_draft.md · kakao_draft.md
```

- **①만 순수 스크립트**로 고정한 이유: 중복 판정·tone 정규화·결측 보완은 "정답이 있는 규칙"이라 코드로 고정하면
  실행할 때마다 동일한 판단이 나온다. 반면 ②~③(문장 생성)은 원장에게 자연스럽게 읽히는 카피가 목적이라
  규칙만으로는 품질을 담보할 수 없어, 사람 대신 Claude가 "규칙을 따르는 판단"을 하는 구간으로 남겨두었다.
  즉 재현성은 "①의 결정론적 정제 + ②③의 고정된 규칙"에서 나오지, 문장을 통째로 캐싱해서 나오지 않는다.
- 실행: `python scripts/clean_keywords.py data/keywords_input.csv` → 생성된 `*.cleaned.csv`를 확인 →
  Claude Code에서 `/insight` → `/generate` → `/review` 순서로 이어서 실행한다.

## 2. 채널별 규칙 (톤 · 길이 · 해시태그 · CTA)

| 항목 | 블로그 | 인스타그램 | 카카오채널 |
|------|--------|------------|------------|
| 역할 | SEO 장기 유입 · 신뢰 구축 | 브랜드 인지 · 감성 공감 | 즉각 행동 유도 · 전환 |
| 길이 | 본문(제목·소제목 제외) **500자 이상** | 캡션 **150자 이내** | **180자 이내** |
| 구조 | 도입(공감) → 소제목 H2 2개 → 사례/수치 → CTA | 공감 한 줄 → 핵심 메시지 한 줄 → 행동 유도 | 이모지 + 핵심 혜택 → 조건 → CTA 링크 |
| 해시태그 | 없음 | **5개 고정**: `#올인원장 #학원관리` + keyword·brand 연관 3개 | 없음 |
| 이모지 | 사용 안 함(신뢰도 저하 우려) | 문장 안에 1개 내외, 과하지 않게 | **1~2개** — 3개 이상은 스팸으로 인식돼 전환율을 떨어뜨림 |
| CTA 문구 | "지금 [브랜드]에서 확인해 보세요" 류, 부드러운 유도 | "저장해두세요", "지금 확인하세요" 류, 즉흥적 | "지금 신청하기", "지금 확인하기 → [링크]" — 명령형 + 링크 필수 |
| tone_hint 반영 방식 | 원본 톤 유지, 정보 구조 안에서 표현 | 감성어로 변환(예: 전문적인 → "확실히 달라졌어요") | 행동유도어로 압축(예: 따뜻한 → 담백한 감사 표현 + CTA) |

**근거 (마케팅·원장 관점)**
- 블로그에 이모지·해시태그를 넣지 않는 이유: 학원 관리 앱을 "검색"해서 들어오는 예비 원장에게는
  정보의 신뢰도가 우선이다. 이모지가 많으면 광고성 글로 인식되어 이탈률이 오른다.
- 인스타 해시태그를 `#올인원장 #학원관리`로 고정하는 이유: 케이스마다 바뀌는 3개(주제·brand 연관)만으로는
  검색 유입이 분산된다. 고정 2개가 채널 전체의 콘텐츠를 하나의 해시로 묶어 열피쌤 커뮤니티 등에서
  브랜드 검색 시 누적 노출이 쌓이게 한다.
- 카카오에 링크를 반드시 넣는 이유: `context/industry-news.md`에 명시된 대로 "링크 없이 텍스트만 있으면
  전환율이 급락"하기 때문이다. 카카오채널은 이미 관심 있는 팔로워 대상이므로, 망설임 없이 바로
  행동할 수 있는 통로(링크)를 제공하지 않으면 채널의 존재 이유(전환)가 사라진다.

## 3. type별 채널 강조 매핑

| type | 블로그 강조점 | 인스타 강조점 | 카카오 강조점 | 근거 |
|------|---------------|---------------|----------------|------|
| 신기능 | 기능 소개 + 사용법 + 도입 전후 비교 | "이런 불편 있으셨죠?" 공감 + 한 줄 요약 | "[신기능] 출시 알림 + 바로 써보기 링크" | 신기능은 기존 회원 재활성화가 목적 — 블로그는 왜 필요한지 설득, 카카오는 이미 아는 사람에게 알리기만 하면 되므로 설명을 최소화 |
| 이벤트 | 이벤트 상세 + 참여 방법 + 혜택 정리 | 혜택 요약 + 기간 강조 + "저장해두세요" | 마감일·선착순 강조 + 신청 링크 | 이벤트는 신규 가입 직접 유도가 목적 — 카카오·인스타에서 기간·정원을 반복 노출해 지금 행동하게 만드는 것이 핵심(4장 데이터 분석 참고) |
| 후기 | 원장/학부모 인터뷰 형식 + 수치 포함 | 후기 인용구 한 줄 + "나도 해봐야겠다" | 후기 핵심 한 줄 + 무료 체험 CTA | 후기는 신뢰 구축이 목적 — `industry-news.md`가 밝힌 2026년 "진정성(Authenticity)" 트렌드에 따라 실제 발화를 그대로 인용하는 것이 과장 광고보다 효과적 |

**tone_hint → 채널별 변형 예시**
| tone_hint | 블로그 | 인스타 | 카카오 |
|---|---|---|---|
| 전문적인 | 수치·기능 스펙 중심 서술 | "확실히 달라졌어요" 같은 결과 중심 한 줄 | 담백한 기능명 + 링크 |
| 따뜻한/친근한 | 원장·학부모 경험담 인용 | 공감 질문형 문장 | 감사 인사 + CTA |
| 밝은 | 이벤트·시즌 기획 의도 설명 | 이모지 + 가벼운 어투 | 혜택 강조 + 이모지 |
| 긴급한 | 기간 한정 사실을 소제목에 명시 | "마감임박" 강조어 | ⏰🔥 등 긴급 이모지 + 날짜 명시 |
| 자랑스러운 | 성과 수치를 소제목화 | 수치를 굵게 강조하는 한 줄 | 수치 + "실제 후기" 문구 |

## 4. 재현성 검증

`scripts/clean_keywords.py`로 임의의 새 CSV(다른 중복·다른 tone 오탈자·다른 결측 패턴)를 넣어
동일한 로직(중복 판정 → tone 정규화 → detail 보완 → type 집계)이 그대로 재현되는 것을 확인했다.
실제 `keywords_input.csv`(25행) 실행 결과는 `data/cleaning_report.md`에 남아 있다 —
25건 → 24건(중복 1건 제거), tone_hint 정규화 1건, detail 보완 2건, type 분포 신기능/이벤트/후기 각 8건.
