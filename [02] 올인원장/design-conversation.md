## 1. 결과물 임팩트 논의

### Q. output이 약하다는 고민 — 구체화 또는 대안 아이디어?

**현황 문제**: 구버전(problem-02-allinone)은 "3채널 콘텐츠 초안 생성"이 1회성 문서로 끝나, AI 활용 역량(채널 전략 설계·재현 파이프라인)이 잘 드러나지 않음

**브레인스토밍 결과 — 세 가지 방향 검토**

| 옵션 | 설명 | 결론 |
|------|------|------|
| A. 채널별 콘텐츠 생성 스크립트/스킬 | `/generate`로 25건 3채널 초안 자동 생성 | Standard 레벨로 포함 |
| B. 카카오채널 자동 발행 연동 | API로 실제 발송 | **제외 범위**로 결정 (노션 ⑥ 카카오채널 API 제외) |
| C. 키워드 패턴 기반 발행 전략 기획 | 25건 패턴 → 채널·시즌·type별 발행 전략 | Challenge 레벨로 포함 |

## 2. 레벨 구조 설계

### Q. 현재 목표를 레벨 1로 하고 레벨 2, 3을 설정한다면?

**설계 원칙**: "1회성 콘텐츠 생성" → "새 키워드에도 동일 품질로 도는 자동화 + 전략 기획"으로 발전하는 누적형 구조 (콘텐츠 생성형 페르소나 사다리: 콘텐츠 마케터 → 마케팅 PM → 콘텐츠 전략가)

### 최종 확정 레벨

```
🟢 Basic (100점)  — 콘텐츠 마케터 수준
  → 25건 키워드를 블로그(500자+)·인스타(150자 이내)·카카오(180자 이내) 3채널 초안 생성
  → 채널별 톤 차별화 + keyword·brand_name 반영
  → 제출: blog/instagram/kakao_draft.md + 노션·구글독스 공개 링크

🟡 Standard (+30점, 누적 130점)  — 마케팅 PM 수준
  → Basic + 재사용 가능한 파이프라인
  → 새 keywords_input.csv 넣으면 정제→채널 전략→3채널 초안 자동 생성 (/명령 또는 스크립트)
  → 채널별 톤·길이·해시태그·CTA 규칙과 type별 강조 매핑을 본인이 직접 설계

🔴 Challenge (+30점, 누적 160점)  — 콘텐츠 전략가 수준
  → Standard + 키워드 패턴에서 콘텐츠 발행 전략 기획안 자동 생성
  → 기획안 구성: 문제 정의 / 근거 데이터 / 발행 제안 / 우선순위
  → 기획안 양식·우선순위 기준도 본인이 설계
```

> 구버전의 단순 배점(블로그30/인스타20/카카오20/톤15/처리10 + 보너스5)을
> [01]과 동일한 누적형 100/+30/+30 체계로 재설계함.

## 3. CSV 데이터 재설계

### Q. 노션 문서(Problem 02 v2)와 로컬 파일 정합성 맞추기

**변경 전 컬럼** (구버전 content-requests.csv, 14행)
```
id, 날짜, 콘텐츠유형, 키워드1, 키워드2, 키워드3, 타깃, 톤, 요청자메모
```

**변경 후 컬럼** (노션 v2 ④ 기준, keywords_input.csv, 25행)
```
id, type, keyword, tone_hint, brand_name, detail
```

**주요 변경 사항**
- 파일명 `content-requests.csv` → `keywords_input.csv` (노션 ④ 명시)
- `날짜` 제거 (콘텐츠 생성에 불필요, 분량만 차지)
- `키워드1/2/3` 3개 컬럼 → `keyword` 1개 컬럼(쉼표 구분 1~3개)로 통합 (노션 ④)
- `콘텐츠유형` → `type`, **값 4종(신기능/이벤트/후기/시즌) → 3종(신기능/이벤트/후기)** 로 축소 (노션 ④ type 3가지 명시 — "시즌" 제거)
- `타깃` 제거 → `brand_name` 추가 (학원 이름 커스텀, 노션 ④). 채널마다 brand가 노출되어야 현실적
- `톤` → `tone_hint` (전문적인/따뜻한/긴급한/자랑스러운/밝은/친근한 등으로 다양화)
- `요청자메모` → `detail` (부가 설명, 선택 — 비어 있을 수 있음)
- 행수 14 → **정확히 25행** (노션 ④ "케이스 수 25개", type 3종 × tone_hint 조합)
- type 분포: 신기능 9 / 이벤트 8 / 후기 8

**의도적 데이터 품질 이슈 (참여자가 처리해야 할 과제)**
- detail 결측 2건: id 012, 020 (빈 셀) — type·keyword·brand_name으로 보완 가능
- 완전 중복 행 1건: id 025 = id 001 (모든 컬럼 값 동일)
- tone_hint 표기 흔들림 1건: id 017 "긴박" (다른 긴급 케이스 008·014는 "긴급한") — 정규화 판단 필요
- ※ 정답 컬럼 없음: 채널별 완성 콘텐츠나 "정답 톤" 같은 라벨은 데이터에 넣지 않음. tone_hint는 입력 힌트라 OK.

## 4. Output 형식 결정

### Q. md 파일이면 안 됨 — 시각적으로 확인할 수 있어야 함

**결론**: 노션 ③ 명시대로 채널별 .md(blog/instagram/kakao_draft.md)를 기본 산출물로 하되,
각 레벨별 HTML 샘플 파일로 시각 검수 가능하게 함

| 파일 | 주요 시각 요소 |
|------|----------------|
| `output/sample-basic.html` | type 필터 버튼 + 케이스 카드(3채널 탭/컬럼) + 글자 수 배지 + 복사 버튼 |
| `output/sample-standard.html` | 파이프라인 플로우 다이어그램 + 채널 전략 규칙표 + type별 강조 매핑 |
| `output/sample-challenge.html` | 우선순위 점수 계산기 + 발행 전략 기획안 카드 + 발행 캘린더/로드맵 |
