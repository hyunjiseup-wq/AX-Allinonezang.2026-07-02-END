# 🎯 해커톤 결과물 발표자료 셀프제작 가이드

> 오늘 해커톤에서 만든 결과물을, **5분 발표용 포트폴리오 슬라이드(16:9)**로 자동 제작해 드립니다.
> 내가 푼 문제·해결 과정·결과물을 입력하면, 채용 담당자가 한눈에 이해하는 HTML 발표 덱이 완성돼요.

---

## 이렇게 하면 됩니다 (5단계)

1. **[claude.ai](https://claude.ai) 접속** → 새 대화 시작 *(무료 플랜 가능, 로그인만 되어 있으면 됩니다)*
2. 아래 **[복사용 프롬프트]를 통째로 복사**해서 대화창에 붙여넣기
3. 프롬프트 맨 아래 `[내 해커톤 작업 내용]` 자리에 **오늘 만든 것, 해결 과정, 결과물을 자유롭게 붙여넣기**
4. 전송 →
   - 내용이 **부족하면** Claude가 빠진 부분을 **하나씩 물어봅니다** → 답하면 채워줌
   - 그런 다음 **완성된 발표 슬라이드를 오른쪽에 미리보기(아티팩트)**로 제작
5. **PDF로 저장:** 아티팩트 우측 상단 메뉴 → HTML 내려받기 → 브라우저로 열기 → `Cmd+P`(윈도우 `Ctrl+P`) → 용지 **A4 + 방향 "가로(Landscape)" + "배경 그래픽" 체크** → PDF로 저장 *(슬라이드 1장 = 1페이지)*

> 신경 쓸 건 **3번(오늘 한 일 입력)** 하나예요. 나머지는 프롬프트가 알아서 구성합니다.

---

### 💡 발표자료에 반드시 들어가야 할 5가지

이 5가지가 있으면 채용 담당자가 "같이 일하고 싶다"고 느낍니다.

| # | 항목 | 핵심 질문 |
|---|------|----------|
| 1 | **오늘 받은 문제** | 기업이 제출한 원문 문제가 정확히 무엇이었나? |
| 2 | **내가 재정의한 문제** | 그 문제의 진짜 페인포인트를 내 언어로 좁히면? |
| 3 | **해결 과정 + 막혔던 지점** | 어떻게 접근했고, 어디서 막혀서 어떻게 돌파했나? |
| 4 | **결과물 (데모)** | 실제로 동작하는 것을 만들었나? 영상/GIF/스크린샷이 있나? |
| 5 | **비즈니스 임팩트** | Before / After 수치로 표현할 수 있는 효과가 무엇인가? |

> 수치가 없어도 괜찮아요. "없는 숫자를 지어내지 말고, 무엇이 어떻게 좋아졌는지"를 구체적으로만 써도 됩니다.

## 📋 [복사용 프롬프트]

아래 회색 박스를 통째로 복사하세요.

````text
너는 팀스파르타 AX 해커톤 참가자의 오늘 결과물을, 아래 고정된 16:9 발표 슬라이드 덱 디자인 템플릿에 맞춰 완성해 주는 발표자료 디자이너야.

[독자 정의 — 이것을 항상 기억해]
이 자료의 독자는 "오늘 문제를 낸 기업의 채용 담당자"야.
① 문제를 누구보다 잘 알고 ② 채용을 검토하며 ③ 바쁘고 ④ 기술 이해도는 낮다.
모든 슬라이드는 "이 담당자가 지금 궁금한 게 뭔가"에 답하도록 구성해.

[슬라이드 구조 — 이 순서 그대로]
슬라이드 0. 표지 — 이름 + 태그라인 ("〈기업 문제〉를 〈무엇으로〉 풀었다")
슬라이드 1. 한 장 요약 — 받은 문제 원문 / 만든 것 한 문장 / 핵심 임팩트 수치 1~2개 / 데모 링크
슬라이드 2. 문제 이해 — 기업 원문 → 내가 재정의한 문제 → Before 수치 → 스코프(푼 것/안 푼 것)
슬라이드 3. 해결 접근 & 의사결정 — 접근 가설 / 버린 대안 + 이유 / 핵심 흐름(비전공자 이해 수준) / ★막혔던 지점 & 돌파 방법★ / AI vs 내 판단
슬라이드 4. 산출물 & 실증 — 데모 영상/GIF 자리 + 링크 / 입력→처리→출력 한 사이클 / "무슨 일을 대신해 주는가"로만 설명(기술명 빼기)
슬라이드 5. 임팩트 — Before/After 정량 표 / 정성 효과 / 수치 근거·가정 명시
슬라이드 6. 적용 가능성 & 아쉬움 & 비전 — 우리 환경 적용 시 필요한 것 / 솔직한 아쉬움 / "2주 더 주어진다면" 확장 방향
슬라이드 7. 마무리 — 이 문제를 계속 맡았을 때의 그림 한 줄 / 이 문제와 나의 연결고리 한 줄
(별첨) 기술&논리 상세 — 워크플로우 구조 / 주요 도구·노드 / 데이터 처리 과정 / 추가 테스트 기록

[작업 지침]
1. 맨 아래 [내 해커톤 작업 내용]을 읽어. 만약 비어 있으면, "오늘 작업 내용을 붙여넣어 주세요"라고만 답하고 멈춰.

2. 위 슬라이드 구조에 맞게 내용을 배치해.

3. [내용이 빠져 있을 때 — 채우도록 유도]
   - 항목 중 빠졌거나 불충분한 것이 있으면, 절대 지어내지 말고 사용자에게 질문해서 채워.
   - 질문은 한 번에 하나씩, 어떤 슬라이드의 어떤 항목이 비었는지 콕 짚어서 물어봐.
     (예: "임팩트 슬라이드에서 Before 수치가 없어요. 지금 이 작업을 사람이 수동으로 하면 얼마나 걸리나요?")
   - 특히 '데모(영상·GIF·스크린샷)'와 'Before/After 수치'가 없으면 적극적으로 물어봐.
   - 사용자가 "모른다/없다"고 하면 수치 없이 정성적으로 채우고 넘어가.

4. [표현 원칙 — 전 슬라이드 공통]
   - 읽는 사람은 기술을 모른다. 도구·모델·노드 이름(n8n, GPT-4o 등)은 본문에서 빼고 별첨으로.
   - 본문은 "무슨 일을 대신해 주는가"로만.
     ❌ "n8n 노드로 자동화" → ✅ "이력서 검토 80분 → 8분으로 자동화"
   - 없는 수치를 지어내지 마. 추정치는 반드시 '예상' '추정'임을 밝혀.
   - 정직한 한계 + 구체적 확장 비전 = 신뢰의 핵심. 완벽한 척은 오히려 감점.

4) 여기서 그치지 않고, 24건 데이터에서 반복되는 패턴 2가지(이벤트 마감압박형 vs 웰컴리워드형,
   출결·알림 후기 3건)를 찾아 전환기여도(40%)·반복빈도(35%)·시즌적시성(25%) 가중치로 우선순위를 매긴
   콘텐츠 발행 전략 기획안(content-strategy-proposal.md)까지 작성
5) "재현성" 주장을 말로만 하지 않기 위해, 원본에 없던 keyword·brand 조합의 더미 CSV를 독립적으로
   2세트(8건·5건, 매번 다른 완전중복·미등록 톤·결측 패턴) 만들어 clean_keywords.py로 실측 검증
   (demo/ 폴더). 두 세트 모두 정제 직후 같은 자리에서 끊김 없이 이어서 Claude Code 대화로 3채널
   초안까지 생성해(총 7건, demo/generated_samples.md), "정제(결정론적 스크립트)"와
   "생성(Claude Code 판단)"이 하나의 파이프라인처럼 이어진다는 것을 실연 가능한 형태로 남김.
   특히 매번 다른 미등록 톤("차분한"→따뜻한 계열, "설레는"→밝은 계열)이 나왔지만 고정 규칙표를
   그대로 찾는 게 아니라 매번 문맥에 맞게 판단하는 것도 확인됨
6) 데이터 정제만 눈으로 빠르게 보여줄 수 있도록, clean_keywords.py 로직을 그대로 옮긴 가벼운
   Streamlit 뷰어(CSV 업로드 → 정제 결과 표시, 외부 AI 호출 없음)를 별도 저장소로 배포.
   과제 제출 저장소와는 분리해, 시연에서는 "정제 확인용 프리뷰"로만 쓰고 실제 3채널 생성은
   항상 Claude Code 대화로 끝나도록 설계

- 막혔던 순간과 어떻게 돌파했는지:
"새 CSV를 넣어도 재현되게 만들라"는 요구사항을 보고, Anthropic API 키를 발급받아 Streamlit으로
자동 생성 봇을 직접 만들었음. 실제로 API 연동까지 끝내 정상 동작을 확인했는데, 그 직후 이 과제의
CLAUDE.md 제약조건("별도 API 키 없음 — Claude Code 대화 안에서 직접 콘텐츠 생성", "웹 UI 구현 제외")을
다시 확인하고 방향이 어긋났다는 걸 깨달음. 봇을 버리는 대신 별도 폴더로 분리해 개인 실험으로 남기고,
"재현성"이라는 원래 요구사항은 스크립트(clean_keywords.py)와 Claude Code 명령(/generate)의 조합으로
푸는 방향으로 되돌림. 결과적으로 "자동화 = 반드시 별도 앱"이라는 전제 자체가 틀렸다는 걸 배움 —
Claude Code 안에서도 스크립트+명령 조합으로 충분히 재현 가능한 파이프라인을 만들 수 있었음.
````
