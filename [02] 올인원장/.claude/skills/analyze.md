# /analyze — 콘텐츠 요청 데이터 파악

## 목적

data/keywords_input.csv를 읽고 콘텐츠 생성 전 데이터 현황을 파악합니다.
결측치, 중복, tone_hint 표기 흔들림, type 분포를 확인해 generate 단계의 예외 처리 방향을 결정합니다.

---

## 실행 절차

### 0단계: 데이터 클렌징 스크립트 실행 (재현 파이프라인의 시작점)
새 keywords_input.csv로 교체했다면 먼저 아래를 실행하세요:
```
python scripts/clean_keywords.py data/keywords_input.csv
```
중복 제거·tone_hint 정규화·detail 결측 보완이 결정론적으로 처리되어 `data/keywords_input.cleaned.csv`와
`data/cleaning_report.md`가 생성됩니다. 이후 단계는 이 정제된 파일을 기준으로 진행합니다.

### 1단계: 파일 읽기
`data/keywords_input.cleaned.csv`(0단계 실행 전이면 원본 `keywords_input.csv`)를 Read 툴로 읽으세요. (총 25행 + 헤더, 정제 후 기준 건수는 cleaning_report.md 참고)

### 2단계: 컬럼 구조 확인
아래 컬럼이 모두 존재하는지 확인하세요:
- id, type, keyword, tone_hint, brand_name, detail

### 3단계: 결측·중복·표기 탐지
- **detail 결측**: 빈 값인 행의 id를 나열하세요
- **완전 중복 행**: 모든 컬럼 값이 동일한 행 쌍을 찾으세요 (id만 다른 경우 포함)
- **tone_hint 표기 흔들림**: 같은 의미인데 어미·표기가 다른 값을 찾으세요 (예: "긴급한" vs "긴박")

### 4단계: type 분포 집계
- 신기능: __ 건
- 이벤트: __ 건
- 후기: __ 건

### 5단계: 채널별 요구사항 정리
| 채널 | 길이 기준 | 비고 |
|------|-----------|------|
| 블로그 | 500자 이상 | 공백 포함, 소제목 H2 2~3개 |
| 인스타그램 | 150자 이내 | 캡션 본문만 (해시태그 5개 별도) |
| 카카오채널 | 180자 이내 | 이모지 1~2개 포함 |

---

## 출력 형식

```
## 데이터 분석 결과

### 전체 건수
- 총 25건 (중복 포함)

### 결측·중복·표기 현황
- detail 결측: id [번호] (__ 건)
- 완전 중복 행: id [번호] = id [번호]
- tone_hint 표기 흔들림: id [번호] — "[원본값]" → "[정규화값]"

### type 분포
- 신기능 __ 건 / 이벤트 __ 건 / 후기 __ 건

### 처리 방향 제안
- detail 결측: [처리 방법]
- 중복 행: [처리 방법]
- tone_hint 정규화: [처리 방법]
```
