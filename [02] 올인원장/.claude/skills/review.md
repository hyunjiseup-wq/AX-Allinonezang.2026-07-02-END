# /review — 콘텐츠 자가 점검

## 목적

output/에 저장된 케이스별 콘텐츠 초안을 검토하여
글자 수 기준 충족 여부, 채널별 톤 차별화, 데이터 처리 정확성을 확인합니다.
CLAUDE.md의 누적형 채점 기준에 맞춰 자가 채점합니다.

---

## 전제 조건
- /generate 완료 후 실행
- output/ 결과 파일이 존재해야 함

---

## 점검 체크리스트

### 1. 글자 수 점검 (Basic 길이 충족 40점)
각 케이스에 대해:
- [ ] 블로그: 500자 이상인가? (실제 본문 글자 수 일치 확인)
- [ ] 인스타: 캡션 본문 150자 이내인가? (해시태그 제외)
- [ ] 카카오: 180자 이내인가? (이모지 포함)

미달/초과 시: 해당 id·채널 명시 → 현재/목표 글자 수 차이 → 보완 방향
(예: "블로그 id#012: 현재 423자 → 77자 추가 필요")

### 2. 채널별 톤 차별화 점검 (Basic 톤 차별화 30점)
- [ ] 블로그: 정보성 구조(도입→소제목H2→CTA)인가?
- [ ] 인스타: 공감형 감성 언어, 짧고 임팩트 있는 문장 + 해시태그 5개인가?
- [ ] 카카오: 행동 유도(CTA) 중심, 간결한 2~3문장 + 이모지 1~2개인가?
- [ ] 같은 문장이 채널 간 그대로 복붙되지 않았는가? (복붙 = 재작성 필요)

### 3. 키워드·brand 반영 점검 (Basic 반영 20점)
- [ ] keyword(1~3개)가 각 채널에 자연스럽게 녹았는가?
- [ ] brand_name(학원명)이 각 채널에 노출되었는가?
- [ ] type(신기능/이벤트/후기)에 맞는 강조점이 적용되었는가?

### 4. 데이터 처리 정확성 점검
- [ ] detail 결측 케이스: 보완 처리가 적용되었는가?
- [ ] 완전 중복 행: 정책대로 처리되었는가?
- [ ] tone_hint 표기 흔들림: 정규화되었는가?
- [ ] decisions.md에 처리 결정이 기록되어 있는가?

### 5. 제출 형식 점검 (Basic 형식 10점)
- [ ] blog_draft.md / instagram_draft.md / kakao_draft.md (또는 통합 1개)로 저장되었는가?
- [ ] 각 케이스 헤더에 id / type / keyword 메타가 있는가?

## 점검 결과 출력 형식

```
## /review 점검 결과

### 글자 수 점검
| id | 블로그 | 인스타 | 카카오 | 판정 |
|----|--------|--------|--------|------|
| 001 | 523자 | 138자 | 172자 | 통과 |
...

### 재작성 필요 항목
- id#[번호]: [채널] — [문제] → [보완 방향]

### 채널별 톤 차별화 / 데이터 처리
- 이슈 없음 / 이슈 있는 id 목록

### 누적형 자가 채점
- Basic: 길이 __/40 + 톤 __/30 + 반영 __/20 + 형식 __/10 = __/100
- (Standard·Challenge 도전 시) 가산 항목 자가 평가

### 전체 통과율
- 길이 기준 통과: __ / 25 건
- 완전 통과 (길이 + 톤 + 처리): __ / 25 건
```

---

## 재작성 처리
- 블로그 500자 미달 → 소제목 단락 확장 (기능 사용 예시·수치 추가)
- 인스타 150자 초과 → 핵심 문장 1개만 남기고 압축
- 카카오 180자 초과 → 첫 문장 + CTA만 남기고 압축

---

## 완료 조건
- 25건 전체 글자 수 점검 완료
- 미달/초과 항목 보완 완료
- 채널별 톤 차별화 확인 완료
- 누적형 자가 채점 결과가 출력됨
