# /generate — 3채널 콘텐츠 초안 생성

## 목적

data/keywords_input.csv의 각 케이스(행)에 대해 블로그·인스타그램·카카오채널 3채널 초안을 생성하고
output/ 폴더에 저장합니다.

---

## 전제 조건

1. /analyze 완료 — 결측·중복·표기 흔들림 파악됨
2. /insight 완료 — 채널별·type별 전략 수립됨
3. `context/company-info.md`, `context/industry-news.md` 읽기 완료

---

## 생성 규칙

### 글자 수 규칙 (필수)
| 채널 | 기준 | 측정 방법 |
|------|------|-----------|
| 블로그 | 500자 이상 | 공백 포함, 소제목 H2 2~3개 필수 |
| 인스타그램 | 150자 이내 | 캡션 본문만 (해시태그 5개 별도) |
| 카카오채널 | 180자 이내 | 이모지 1~2개 포함 |

### 채널별 톤 차별화 규칙
각 행의 `tone_hint`를 기반으로 3채널에 각각 적용하되, 채널 특성에 맞게 변형하세요:
- 블로그: tone_hint 유지 + 정보성 글 구조(소제목 H2)
- 인스타: tone_hint를 감성·공감 언어로 변환
- 카카오: tone_hint를 간결한 행동 유도(CTA) 언어로 변환
- **같은 키워드라도 3채널이 그대로 복붙되면 안 됨**

### keyword·brand_name 반영
- keyword(1~3개)를 각 채널에 자연스럽게 녹임
- brand_name(학원 이름)을 각 채널 콘텐츠에 노출
- type(신기능/이벤트/후기)에 맞는 강조점 적용 (/insight의 type별 전략 참고)

### 데이터 예외 처리
- detail 결측: type·keyword·brand_name으로 보완, 추론 내용은 [추론: ...] 표시
- 완전 중복 행: /insight에서 정한 정책대로 처리
- tone_hint 표기 흔들림: 정규화한 톤으로 생성

---

## 생성 순서
1. id 001번부터 순서대로 처리
2. 각 케이스에 대해 아래 출력 형식으로 3채널 초안 생성
3. 생성 후 글자 수를 실제로 세어 기재
4. 25건 모두 완료 후 output/에 저장

---

## 출력 형식 (output/template.md 참고)

케이스별 통합 또는 채널별 분리 모두 가능. 케이스 헤더에 **id / type / keyword 메타 필수**.

```markdown
## 케이스 #[id] · [type] · [keyword]
**brand_name**: [학원명] | **tone_hint**: [톤] | **detail**: [부가설명 또는 결측]

### 블로그 (___자)
**제목**: [SEO 최적화 제목]
## [소제목 H2-1]
[본문]
## [소제목 H2-2]
[본문]
[CTA]

### 인스타그램 (___자)
[캡션 — 150자 이내]
#해시태그1 #해시태그2 #해시태그3 #올인원장 #학원관리

### 카카오채널 (___자)
[메시지 — 180자 이내, 이모지 1~2개] → [링크]
```
